# Инструкции для leaders
- Использовать исключительно веб-интерфейса GitHub.
- Сделать Fork: Каждый лидер делает форк (ответвление) основного репозитория в свою учетную запись GitHub.

## Протокол выбора (используя Issues):
- Перейдите на вкладку "Issues" (Задачи) в основном репозитории (не в вашем форке) и откройте ветку "Student Assingments" (Официальная ветка распределия).
- Объявите о своем выборе комментарием: "Я, [Имя], забираю людей [Имя/Имена]".
- Дождитесь визуального подтверждения, что никто другой не забронировал их непосредственно перед вами. Подтверждение через Issue — важное правило.
## Редактируете файл team_assignment.md в Веб-интерфейсе (в своем форке):
- Перейдите в свой собственный форк (ваша версия репозитория).
- Перейдите к файлу team_assignment.md.
- Нажмите на значок карандаша (редактировать этот файл) в правом верхнем углу просмотра файла.
- Добавьте имена людей, которых вы забронировали в Issue.
- Нажмите "Commit changes" (Зафиксировать изменения) внизу страницы.
## Создать Pull Request (PR) через Веб:
- После фиксации изменений (commit) GitHub предложит создать Pull Request. Если нет, перейдите на вкладку "Pull Requests" в вашем форке и нажмите "New pull request".
- Убедитесь, что базовым репозиторием является оригинал, а сравниваемым — ваш форк.
- Опишите изменения (например, "Назначаю 5 человек в свою команду") и отправьте PR.
## Синхронизировать форк 
- Перед попыткой сделать новый выбор или редактирование лидер должен синхронизировать свой форк с основной веткой оригинального репозитория.
- На главной странице вашего форка найдите кнопку "Fetch upstream" и выберите "Update branch". Это подтянет изменения, которые владелец репозитория уже объединил от других лидеров (шаг важен, чтобы видеть назначения других).
